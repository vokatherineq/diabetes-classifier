Created by - Parallel Processing Team

What does the project do?

Our goal for this project is to be able to create a model that can identify whether or not a person has diabetes. By using a data set that includes several health levels such as: Glucose, Insulin, and Age, our team is able to train our model to identify the symptom levels of diabetes.

Exploration of data: Heatmap

A heatmap's main purpose is to display the correlation between features in a data set. If you look below you can see all the features: Glucose, BMI, etc. and their correlation to the outcome.

Glucose and BMI were the two most correlated factors to whether or not a person has diabetes.

Exploration of Data: Boxplots

The boxplots below can be used to sift through all the data and any points outside of the range will be determined as faulty data with the exception of age. The rectangle in the range represents a broad average and the line inside of the rectangle represents the true average.

The boxplots were one of the most useful models we made, because it helped us find outliers. once we found the outliers, we dropped those values from our dataset which resulted in the accuracy of our model increasing.

Exploration of Data: Scatterplots

Scatter plots mainly show the correlation between 2 variables in a data set, in this case we are showing the correlation count between the 'Outcome', and a specific health value (x, y). This is evident in the second scatter plot, we were able to see that people with a high glucose level usually also had diabetes.

Some of the x variables had a positive correlation such as blood pressure, thick skin, and insulin. Because of this, we want to include this in order model to see if it can help predict the chance of diabetes.

Machine Learning Models





Logistic Regression

Logistic Regression is used when predicting an outcome based on a fixed variable such as determining possible risk of disease using health factors like Glucose level and BMI (Body Mass Index). Before predicting an outcome, filtering through data is usually the best course of action because it allows faulty data to be discarded such as if the data set displays someone’s blood pressure as 0 or 400.

Logistic Regression accuracy - 84.6 %

Confusion Matrix for Logistic Regression

K-nearest neighbors

KNN takes a data point and find the k nearest points to that data point. If k=5, KNN find the 5 nearest neighbors and chooses the class of neighbor that has the majority. This is why k should not be an even number because there is a possibility of a tie. In this data set I tested many different k values and found that k=47 gave the best accuracy.

K-nearest neighbors accuracy - 85.9 %

Confusion Matrix for k-Nearest Neighbors

Random Forests

Random forest works by constructing a “forest” of decision trees and outputting the average result from the trees to categorize data. To increase the accuracy of the method, we can go through the data set and take out outliers and only apply the most relevant data points.

Random Forest accuracy - 80.7 %

Confusion Matrix for Random Forests





-Conclusion-

Based on the results, our team has come to the conclusion that k-nearest neighbors has performed the best compared to all the ML models. With cleaning the data and handling the outliers, all models were able to get to significantly high accuracy points:

Logistic Regression accuracy - 84.6 %

K-nearest neighbors accuracy - 85.9 %

Random Forest accuracy - 80.7 %

Since our data set is not the most significant(big) and recent set, further research must be done in order to use this model in the real world. However with more recent research and data this model could be able to prevent people from experiencing harsh effects of untreated diabetes, along with helping people try and reverse it. If our model is able to point out diabetes early enough, then the individual has a chance of reversing their condition.
